## Project Overview

**Unscripted Living** — A personal retirement blog covering hobbies (woodworking, photography, reading, guitar, IT), daily life, and thoughts on finance, environment, and AI.

**Status:** Active development

**Stack:**
- Python 3.12 / Django 6.0
- PostgreSQL (shared k8s instance)
- django-allauth (email + Google OAuth)
- Bootstrap 5 + custom CSS (clean card-based design)
- WhiteNoise for static files
- OpenTelemetry for observability
- Docker + Kubernetes (namespace: `unscripted-living`)
- Package manager: uv

## Development Setup

```bash
# Install dependencies
uv sync

# Set up environment
cp .env.example .env  # fill in real values

# Run migrations
uv run python manage.py migrate

# Create superuser
uv run python manage.py createsuperuser

# Run the dev server
uv run python manage.py runserver

# Add a package
uv add <package-name>

# Run a specific migration
uv run python manage.py migrate blog

# Check for issues
uv run python manage.py check
```

**Environment variables:** Copy `.env.example` to `.env`. Database host/port for local dev uses the k8s external address. See `documents/setup/05_postgres_setup.md`.

## Architecture

Single Django app (`blog/`) with Category → Post → Comment hierarchy. All blog content is public by default; comments require authentication. The homepage is driven by `HomeView` which queries featured + recent published posts and all categories. Categories inject into every page via the `blog.context_processors.nav_categories` context processor.

Post content is stored as Markdown and rendered in `PostDetailView` using `markdown` + `bleach` for safe HTML output.

### Key entry points
- `config/urls.py` — root URL config (admin + allauth + blog)
- `blog/views.py` — all blog views
- `blog/models.py` — Category, Tag, Post, Comment models
- `config/settings.py` — all configuration (reads from `.env`)
- `config/middleware.py` — JSON request/response logging

### Non-obvious conventions
- All credentials come from `.env` (local) or k8s SealedSecrets (production). Never hardcode.
- `blog/context_processors.py` injects `nav_categories` into every template — no need to pass categories from individual views.
- Post `published_at` is set automatically on first save when `status='published'`.
- The `temp.yaml` in `k8s/` is gitignored plain-text secrets for sealing — delete after running kubeseal.

### Files to never edit directly
- `staticfiles/` — generated by `collectstatic`
- `media/` — user uploads
- `blog/migrations/` — generated by `makemigrations`, don't hand-edit

## Deployment

```bash
# Build and push multi-arch Docker image
docker buildx build --platform linux/amd64,linux/arm64 -t jaysuzi5/unscripted-living:latest --push .

# Apply k8s manifests
kubectl apply -f k8s/deployment.yaml

# Restart deployment
kubectl rollout restart deployment unscripted-living -n unscripted-living

# Check pods
kubectl get all -n unscripted-living

# Exec into pod
kubectl exec -n unscripted-living -it <pod-name> -- /bin/sh
```

**Domain:** unscripted.jaycurtis.org (Cloudflare tunnel)

**Secrets:** Managed as SealedSecrets in `k8s/secrets.yaml`. See `documents/setup/05_postgres_setup.md` for sealing instructions.
